What is Elasticsearch?

  • A near-real-time application for quickly indexing and searching 'documents'
  • Node / Cluster system that scales to the web
  • Stores documents as an inverted index to find what you need, fast
  • Stores _source data alongside the index for fast retrieval
  • Apache Lucene under the hood
  • First built as a recipe search, now a full-fledged company + support

 

 

What's a reverse index?

Things I like about Elasticsearch:

  • REST interface for all the things
  • Use with any language because HTTP (Yay!)
  • It's fast and works well with JavaScript (I'm biased)
  • Clients for most languages available (more on this later)
  • Really easy to get up and running!
  • Really powerful, and easy to extend
  • Open source from the start
  • Solid docs.

A few Nifty and Notable Features:

  • Full Text search with Relevance
  • Text analyzers (grammar, root word, pluralization, etc)
  • "More Like This" Term Frequency-based Queries
  • Advanced Queries with operators (fuzzy search, phoenetic, etc)
  • Aggregations of all kinds (including analytics / stats)
  • Geospatial Queries / Aggregations

Why Use Elasticsearch?

  • You know, for search!
  • Logs / Time series capture
  • Rapid filtering / counts / aggregation of data
  • Big Data (or, you know, sizeable data)
  • Supplement existing Big Data efforts (Spark, Kafka, etc)
  • Percolation / Alerting
  • Data Vizualization w/ Kibana

The Hard Bits

  • Getting started is easy. Maturing to your business case is harder.
  • A good deal of gotchas and lots of best practices to learn
  • Gotcha factor is getting better but still takes time to learn the inner workings / API
  • Using the API is straightforward. Getting your data ready and thinking about structure is hard.

Let's Get Cracking!
